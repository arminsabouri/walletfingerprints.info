Wallet Fingerprints

On-chain transaction fingerprints across payjoin integrations

// context

Every Bitcoin wallet leaves a fingerprint. When software constructs a transaction it makes dozens of small decisions — input ordering, locktime values, signature encoding, fee estimation — that vary systematically across implementations. These patterns are observable on-chain and allow an adversary to identify which wallet created a transaction, even without any off-chain data.

For collaborative transactions like PayJoins, fingerprints are especially damaging. If the sender and receiver use different wallet software, signature and script differences partition the inputs by owner — collapsing the privacy back to a pair of regular transactions. Even when intra-transaction signals are uniform, the analyst can look backward and forward in the transaction graph where each wallet's fingerprints reappear.
